# 家庭日历聚合与家务助手

现代家庭要处理五个或更多日历——工作、个人、共享家庭、孩子的学校、课外活动——跨不同平台和格式。重要事件会被遗漏，因为不存在单一视图。与此同时，家务协调（购物清单、储藏室库存、预约安排）通过零散的短信进行，这些短信会被埋没。

这个用例将 OpenClaw 变成一个始终在线的家务协调员：将日历聚合到晨报中，监控消息以获取可操作项目，并通过共享聊天界面管理家务后勤。

## 痛点

- **日历碎片化**：工作日历有安全限制，无法共享。学校日历以 PDF 或手写网站的形式到达。营地时间表存在于电子邮件中。每天早上手动检查每一个是不可持续的——而且"跨日历复制事件效果很好，直到我忘记并且有一个漏掉。"
- **家务协调开销**："我们有多少牛奶？"需要实际检查冰箱，然后是地下室储藏室，然后发短信回复。将此乘以一周的购物次数。
- **错过预约**：预约确认通过短信到达并停留在那里未采取行动——没有日历事件，没有驾驶时间缓冲，没有提醒。

## 功能说明

- **晨报**：将所有家庭日历聚合到通过你首选渠道发送的单一每日摘要中
- **环境消息监控**：监视 iMessage/短信对话，并在检测到预约时自动创建日历事件（牙医确认、会议计划等）
- **驾驶时间缓冲**：在检测到的预约之前和之后添加旅行时间块
- **家务库存**：维护储藏室/冰箱物品的运行库存，任何一方都可以从任何地方查询
- **购物协调**：跨食谱去重配料，跟踪库存不足的物品，并生成购物清单
- **基于照片的输入**：拍摄学校日历或冰箱内容的照片，智能体将其处理成结构化数据

## 所需技能

- 日历 API 访问（Google Calendar、通过 `ical` 的 Apple Calendar）
- `imessage` 技能用于消息监控（仅 macOS）
- Telegram 或 Slack 用于共享家庭聊天界面
- 文件系统访问用于库存跟踪
- 相机/照片处理用于物理日历的 OCR

## 设置方法

### 1. 日历聚合

配置 OpenClaw 从所有家庭日历源提取：

```text
## 日历源

在晨报（早上 8:00）时：

1. 获取我的 Google 工作日历（只读 OAuth）
2. 获取共享家庭 Google 日历
3. 获取伴侣的日历（共享视图）
4. 检查 ~/Documents/school-calendars/ 是否有新的 PDF → OCR 并提取事件
5. 检查最近的电子邮件是否有日历附件或事件邀请

编译成单一简报：
- 今天的事件（所有日历，按来源颜色编码）
- 未来 3 天的前瞻以查找冲突
- 自昨天以来添加的任何新事件
- 户外活动的天气背景

通过 Telegram/Slack 家庭频道发送。
```

### 2. 环境消息监控

这是关键区别——智能体被动监视并在识别到可操作内容时采取行动：

```text
## 消息监控（HEARTBEAT.md）

每 15 分钟：
1. 检查所有对话中的新 iMessage
2. 检测类似预约的模式：
   - "您的预约已确认..."
   - "我们可以在 [日期] [时间] 见面吗？"
   - "练习改到周六下午 3 点"
3. 检测到时：
   - 使用详细信息创建日历事件
   - 在之前和之后添加 30 分钟驾驶缓冲
   - 向家庭 Telegram 发送确认："已创建：牙医预约，周二下午 2 点。添加了驾驶时间 1:30-2:00 和 3:00-3:30。"
   - 如果与伴侣相关，添加邀请
4. 检测承诺/承诺模式：
   - "我会在周五之前发送"
   - "下周一起吃晚饭"
   → 创建日历保留或提醒
```

### 3. 家务库存

```text
## 储藏室跟踪

维护 ~/household/inventory.json，包含：
- 物品名称、数量、位置（冰箱/储藏室/地下室）
- 上次更新时间戳
- 低库存阈值

更新方法：
- 照片：用户发送冰箱/储藏室照片 → 视觉模型提取物品
- 文本："我们没有鸡蛋了" / "买了 2 加仑牛奶"
- 收据：购物收据照片 → 更新库存

查询：任何一方都可以通过 Telegram 询问：
- "我们有黄油吗？" → 检查库存，回复位置和数量
- "什么库存不足？" → 列出低于阈值的物品
- "生成购物清单" → 编译低库存物品 + 所需的任何食谱配料
```

## 关键见解

- **环境 > 主动**：最大的解锁是智能体在不被要求的情况下采取行动。在短信中检测预约并创建带有驾驶缓冲的日历事件——"我没有要求它这样做。它只是知道这是我想要的。"
- **Mac Mini 是最佳选择**：这个用例从在家用 Mac Mini 上运行中受益匪浅——iMessage 集成、Apple Calendar 和始终在线可用性
- **从只读开始**：在启用写入操作（创建事件、发送消息）之前，从日历读取和消息监控开始
- **共享 Telegram 频道**：让双方都能看到智能体在做什么——建立信任并及早发现错误
- **照片输入被低估**：拍摄学校日历 PDF 或冰箱内容的照片比打字更快——而且视觉模型处理得很好

## 灵感来源

这个用例结合了几个社区模式：

- **日历聚合**：由 HN 用户 `angiolillo` 在 [Hacker News 讨论](https://news.ycombinator.com/item?id=46872465) 中描述，他详细说明了每天早上分别检查工作、个人、家庭和孩子学校日历的痛苦。
- **环境消息监控**：由 [Sparkry AI](https://sparkryai.substack.com/p/24-hours-with-openclaw-the-ai-setup) 记录——当妻子收到牙科预约短信时，OpenClaw 自动创建了一个带有 30 分钟驾驶缓冲的日历事件，而无需被要求。也在 [OpenClaw Showcase](https://openclaw.ai/showcase) 上得到确认，其中 `@theaaron` 称基于聊天的日历管理为"我体验过的 LLM 最佳用途之一"。
- **家务协调**：Brandon Wang 的 [Clawdbot "Linguini"](https://brandon.wang/2026/clawdbot) 在家中的 Mac Mini 上运行——处理短信跟进、从照片创建日历事件、跟踪 Airbnb 价格、处理冰箱库存照片，并通过 iMessage 和 Slack 协调家务后勤。
- **储藏室跟踪**：多个 HN 用户讨论了维护家务库存的价值（和挑战），`dns_snek` 指出："我 5 秒前忘记把东西放在哪里了...这对我来说真的是一个大问题，因为我让东西过期了。"
