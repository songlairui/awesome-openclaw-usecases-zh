# 自动化会议记录与行动项

你刚结束了一个 45 分钟的团队电话会议。现在你需要写总结，提取行动项，并将它们分发到 Jira、Linear 或 Todoist——手动操作。等你完成时，下一个会议已经开始了。如果你的智能体在会议记录到达的那一刻就处理所有这些呢？

这个用例将任何会议记录转换为结构化笔记，并自动在你的项目管理工具中创建任务。

## 痛点

会议记录很繁琐但很关键。大多数人要么跳过它们（并失去上下文），要么花 20 多分钟写它们。行动项被遗忘，因为它们存在于某人的脑海中或埋在聊天线程中。这个智能体消除了"我们讨论了它"和"它被跟踪和分配"之间的差距。

## 功能说明

- **监视** 新的会议记录（通过 Otter.ai 导出、Google Meet 记录、Zoom 录音摘要，或简单地粘贴到聊天中）
- **提取** 关键决策、讨论主题和带有负责人和截止日期的行动项
- **创建任务** 在 Jira、Linear、Todoist 或 Notion 中——分配给正确的人并附带会议上下文
- **发布摘要** 到 Slack 或 Discord，以便整个团队都有记录
- **跟进** — 可选地在截止日期前通过计划提醒 ping 受让人

## 所需技能

- Jira、Linear、Todoist 或 Notion 集成（用于任务创建）
- Slack 或 Discord 集成（用于发布摘要）
- 文件系统访问（用于读取记录文件）
- 调度 / cron（用于跟进提醒）
- 可选：Otter.ai、Fireflies.ai 或 Google Meet API 用于自动记录检索

## 设置方法

1. 选择你的记录来源。最简单的方法是将记录直接粘贴到聊天中。对于自动化，设置文件夹监视或 API 集成。

2. 提示 OpenClaw：
```text
我刚完成了一个会议。这是记录：

[粘贴记录或指向文件]

请：
1. 写一个简洁的摘要（最多 5 个要点），涵盖关键决策和主题。
2. 提取所有行动项。对于每一个，识别：
   - 需要做什么
   - 谁负责（将名字与我的团队匹配）
   - 截止日期（如果提到，否则标记为"待定"）
3. 为每个行动项创建一个 Jira 工单，分配给正确的人。
4. 将完整摘要发布到 Slack 的 #meeting-notes。
```

3. 对于完全自动化的流水线（记录文件夹监视）：
```text
设置一个重复任务：每 30 分钟检查 ~/meeting-transcripts/ 是否有
新的 .txt 或 .vtt 文件。当你找到一个时：

1. 将记录解析为带有行动项的结构化摘要。
2. 在 Linear 中为每个行动项创建任务。
3. 将摘要发布到 Slack 的 #team-updates。
4. 将处理过的文件移动到 ~/meeting-transcripts/processed/。

对于每个有截止日期的行动项，设置提醒在截止日期前一天
在 Slack 中 ping 受让人。
```

4. 自定义输出格式：
```text
在写会议摘要时，始终使用此结构：
- **日期和与会者** 在顶部
- **关键决策** — 决定内容的编号列表
- **行动项** — 表格，列：任务、负责人、截止日期、状态
- **未解决问题** — 任何需要跟进的未解决问题
```

## 关键见解

- 真正的价值不在于摘要——而在于 **自动任务创建**。不会变成跟踪任务的会议记录只是文档表演。
- 将此与 [Todoist 任务管理器](todoist-task-manager.md) 用例配对，以完全了解智能体创建的任务。
- 来自 Zoom 或 Google Meet 的 VTT/SRT 字幕文件作为输入效果很好——它们包含时间戳，有助于智能体将陈述归因于发言人。
- 从简单开始（粘贴记录，获取摘要）并逐步自动化。在验证输出质量之前不要过度设计流水线。
