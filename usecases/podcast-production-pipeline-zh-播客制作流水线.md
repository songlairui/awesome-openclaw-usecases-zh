# 播客制作流水线

你有一个播客创意，甚至可能有一个剧集主题的待办事项列表。但在研究嘉宾、撰写大纲、起草介绍、生成节目笔记和撰写社交媒体推广帖子之间——制作开销扼杀了你的动力。如果你交出一个主题并获得完整的制作包呢？

这个用例将智能体串联在一起，处理从主题到发布就绪资产的整个播客制作工作流。

## 痛点

独立播客和小团队在制作上花费的时间比实际录制更多。研究需要数小时，节目笔记是事后想法，社交媒体推广是第一个被跳过的事情。创意部分——对话——可能只占总工作量的 30%。这个智能体处理其他 70%。

## 功能说明

- **剧集研究** — 给定主题或嘉宾姓名，编译背景研究、谈话要点和建议问题
- **大纲和脚本** — 生成带有介绍脚本、片段过渡和结束语的结构化剧集大纲
- **节目笔记** — 录制后，将记录处理成带有时间戳的节目笔记，并附带提到的所有内容的链接
- **社交媒体套件** — 为 X、LinkedIn 和 Instagram 创建带有剧集亮点和引用的推广帖子
- **剧集描述** — 为 Spotify、Apple Podcasts 和 YouTube 撰写 SEO 优化的剧集描述

## 所需技能

- 网络搜索/研究技能（用于嘉宾研究和主题深入研究）
- 文件系统访问（用于读取记录和写入输出文件）
- Slack、Discord 或 Telegram 集成（用于交付资产）
- 可选：`sessions_spawn` 用于并行运行研究和写作智能体
- 可选：RSS 源技能（用于监控竞争对手播客）

## 设置方法

1. 录制前——生成研究和大纲：
```text
我正在录制关于 [主题] 的播客剧集。我的嘉宾是 [姓名]。

请：
1. 研究嘉宾——他们的背景、最近的工作、热门观点，以及
   他们公开说过的任何有争议或有趣的事情。
2. 研究主题——关键趋势、最近新闻、常见误解，
   以及观众可能已经知道的内容与会让他们惊讶的内容。
3. 生成剧集大纲：
   - 冷开场钩子（1-2 句话吸引注意力）
   - 介绍脚本（30 秒，随意语气）
   - 5-7 个采访问题，从简单/建立融洽关系到深入/挑衅性排序
   - 2-3 个"备用"问题，以防对话停滞
   - 带有行动号召的结束片段

将所有内容保存到 ~/podcast/episodes/[剧集编号]/prep/
```

2. 录制后——生成节目笔记和推广：
```text
这是剧集 [编号] 的记录：[粘贴或指向文件]
```
