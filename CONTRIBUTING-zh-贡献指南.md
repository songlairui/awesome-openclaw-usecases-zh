# 贡献指南

## 添加用例

1. 在 `/usecases` 文件夹中创建一个 markdown 文件，文件名为你的用例名称。
2. 详细描述你的用例。建议包含以下几点：
   - **痛点** — 解决什么问题？
   - **功能说明** — 简要说明用例的作用
   - **提示词** — 给 OpenClaw 的实际提示词或配置
   - **所需技能** — 需要哪些 OpenClaw 技能
   - **相关链接** — 相关文档、API、工具
3. 在主 [README.md](README.md) 的相关分类表格中添加一行
4. 提交 PR

## 准则

- 每个 markdown 文件只包含一个用例
- 描述要简洁，但要足够让别人能够复现
- 如果你的用例不适合现有分类，在 PR 中建议新分类
- 如果方法有明显不同，允许重复的用例
- 不要用 AI 编造新用例。只提交你实际测试和验证过的用例。
- **不接受加密货币相关用例** — 这类用例不会被接受。
